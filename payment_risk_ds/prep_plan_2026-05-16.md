## Phase 4: Interview Prep

### SQL

重点题型：
- transaction-level fraud rate by cohort。
- velocity features: count/sum in last 1h/24h/7d。
- rolling windows。
- false positive / chargeback analysis。
- funnel: attempted payment -> authorized -> captured -> disputed -> charged back。
- user/device/card/IP shared entity analysis。

### Python / ML

重点题型：
- imbalanced classification。
- precision/recall and threshold tradeoff。
- feature engineering for transaction risk。
- time-based train/test split。
- calibration。
- model drift。
- simple graph feature extraction。

### Product / Business Cases

必练：
- How would you detect account takeover?
- How would you reduce payment fraud without hurting conversion?
- How do you evaluate a new fraud rule?
- How do you design a manual review queue?
- What metrics would you track for a payment risk system?
- A fraud model catches more fraud but increases false positives. How do you decide launch?
- How do you identify emerging fraud patterns with delayed labels?

### Domain Knowledge

需要掌握：
- Card-not-present fraud
- Chargebacks / disputes
- First-party fraud / friendly fraud
- Account takeover
- Synthetic identity
- KYC / KYB / OFAC basics
- ACH/wire/card payment basics
- Authorization, capture, settlement, refund
- Manual review and step-up verification
- Fraud labels and delayed feedback
